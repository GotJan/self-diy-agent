## 二、技术栈

| 层 | 技术 | 说明 |
|---|------|------|
| 桌面壳 | Electron 42 | 主进程纯 JS，Vite 只负责渲染进程 |
| 前端 | React 18 + Vite 6 | TypeScript，端口 5173 |
| 开发编排 | concurrently + wait-on | Vite 就绪后自动启动 Electron |
| 状态推送 | SSE (Server-Sent Events) | 零依赖，单向推送 + spec 变更实时通知 |
| 后端 | Express (Node.js) | 端口 3001 |
| Agent | LangChain.js | StateGraph 手动循环 + 动态工具 |
| LLM | DeepSeek v4 Flash | OpenAI 兼容协议，思考模式 |
| 数据库 | sql.js (SQLite WASM) | 纯 JS，无需原生编译，Electron 端 + 后端各一份 |
| 打包 | electron-builder | NSIS 安装器，ASAR + smartUnpack |

### 架构原则

- **Electron 主进程不经过 Vite 编译** — 纯 CJS，`require('electron')` 直接拿到内置 API
- **Vite 只管渲染进程** — 职责单一，不碰 Electron
- **`package.json` 不设 `"type"`** — 默认 CJS，无 ESM/CJS 混用问题
- **`asar.smartUnpack: true`** — 原生模块自动解压

## 三、启动方式

### 1. 前置条件

- Node.js >= 24
- npm

### 2. 配置 API Key

```bash
cd agent-backend
cp .env.example .env
# 编辑 .env，填入 DeepSeek API Key
```

`.env` 内容示例：
```
OPENAI_API_KEY=sk-xxxxxxxxxxxxxxxx
OPENAI_BASE_URL=https://api.deepseek.com/v1
OPENAI_MODEL=deepseek-v4-flash
DEEPSEEK_REASONING_EFFORT=high
```

### 3. 仅启动后端（Agent 服务）

```bash
cd agent-backend
npm install
npx tsx src/index.ts
```

后端启动日志：
```
[SQLite] Connected: .../data/app.db
[Init] tools 表已就绪
[Tool Registry] Registered: create_anime_sprite
...
[Init] 共加载 18 个工具
✅ Agent 已运行，等待输入...
```

### 4. 启动 Electron 开发模式（推荐）

```bash
# 在项目根目录
npm install
npm run electron:dev
```

这会并行启动两个进程：
- `vite` → Vite 开发服务器 `http://localhost:5173`（HMR 热更新）
- `wait-on` 等待 Vite 就绪 → 自动启动 `electron .`

Electron 窗口打开后会自动连接 Vite dev server，并 spawn 后端子进程。

### 5. 仅启动纯前端（无 Electron）

```bash
npm run dev
```

浏览器打开 `http://localhost:5173`。

### 6. 构建 & 打包

```bash
# 仅构建前端
npm run electron:build

# 构建后端 + 前端 + 打包 NSIS 安装器
npm run electron:dist
```

产出在 `releases/Self-Diy-Agent Setup x.x.x.exe`。

## 四、API 端点

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/health` | 健康检查 |
| GET | `/api/status/stream` | SSE 状态推送（启动进度） |
| GET | `/api/specs/watch` | SSE spec 变更推送（创建/修改/删除） |
| POST | `/api/specs/set-initial-state` | 设置 spec 初始状态 |
| GET | `/api/tools` | 查看已注册工具列表 |
| POST | `/api/tools/register` | 动态注册工具 |
| POST | `/api/chat` | 对话（非流式） |
| POST | `/api/chat/stream` | 对话（SSE 流式） |

### SSE 事件类型

```typescript
{ type: 'reasoning', content: '...' }     // 思考过程
{ type: 'tool_call', toolName, args }       // 工具调用
{ type: 'tool_result', toolName, result }   // 工具结果
{ type: 'answer', content: '...' }          // 最终回答（含 token 前缀）
{ type: 'error', message: '...' }           // 错误
{ type: 'done' }                            // 结束
```

最终回答格式：`[2,347/1M Token] 今天是2026年6月12日...`
